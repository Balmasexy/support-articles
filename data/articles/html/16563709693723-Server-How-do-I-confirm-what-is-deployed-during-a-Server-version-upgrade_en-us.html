Verifying Server 4.x version upgrades

This article shares a quick tip on how Server 4.x admins can check what is deployed during a Server 4.x version upgrade.

 

Tip: Use --debug

When upgrading a Server 4.x via Helm, you can include the `--debug` option.
This prints out the verbose outputs which shows which Kubernetes deployments are redeployed or skipped (when there are no changes).

helm upgrade circleci-server \
  oci://cciserver.azurecr.io/circleci-server \
  -n $namespace \
  --version <version> \
  -f <path-to-values.yaml> \
  --debug

 

Note that this tip is applicable only during the upgrade itself.


If you would like to confirm what will be changed with the new version before applying, please see our documentation on how to use the Helm Diff plugin.
